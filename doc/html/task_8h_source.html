2084  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS ONLY

2091  * a ready list. If a non-zero value is returned then a context switch is
2092  * required because either:
2093  * + A task was removed from a blocked list because its timeout had expired,
2094  * or

2100 /*
2101  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
2102  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
2103  *
2104  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
2105  *
2106  * Removes the calling task from the ready list and places it both
2107  * on the list of tasks waiting for a particular event, and the
2108  * list of delayed tasks. The task will be removed from both lists
2109  * and replaced on the ready list should either the event occur (and
2110  * there be no higher priority tasks waiting on the same event) or
2111  * the delay period expires.
2112  *
2113  * The 'unordered' version replaces the event list item value with the
2114  * xItemValue value, and inserts the list item at the end of the list.
2115  *
2116  * The 'ordered' version uses the existing event list item value (which is the
2117  * owning tasks priority) to insert the list item into the event list is task
2118  * priority order.
2119  *
2120  * @param pxEventList The list containing tasks that are blocked waiting
2121  * for the event to occur.
2122  *
2123  * @param xItemValue The item value to use for the event list item when the
2124  * event list is not ordered by task priority.
2125  *
2126  * @param xTicksToWait The maximum amount of time that the task should wait
2127  * for the event to occur. This is specified in kernel ticks,the constant
2128  * portTICK_PERIOD_MS can be used to convert kernel ticks into a real time
2129  * period.
2130  */
2131 void vTaskPlaceOnEventList( List_t * const pxEventList, const TickType_t xTicksToWait ) PRIVILEGED_FUNCTION;
2132 void vTaskPlaceOnUnorderedEventList( List_t * pxEventList, const TickType_t xItemValue, const TickType_t xTicksToWait ) PRIVILEGED_FUNCTION;

2147 /*
2148  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
2149  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
2150  *
2151  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
2152  *
2153  * Removes a task from both the specified event list and the list of blocked
2154  * tasks, and places it on a ready queue.
2155  *
2156  * xTaskRemoveFromEventList()/xTaskRemoveFromUnorderedEventList() will be called
2157  * if either an event occurs to unblock a task, or the block timeout period
2158  * expires.
2159  *
2160  * xTaskRemoveFromEventList() is used when the event list is in task priority
2161  * order. It removes the list item from the head of the event list as that will
2162  * have the highest priority owning task of all the tasks on the event list.
2163  * xTaskRemoveFromUnorderedEventList() is used when the event list is not
2164  * ordered and the event list items hold something other than the owning tasks
2165  * priority. In this case the event list item value is updated to the value
2166  * passed in the xItemValue parameter.
2167  *
2168  * @return pdTRUE if the task being removed has a higher priority than the task
2169  * making the call, otherwise pdFALSE.
2170  */
